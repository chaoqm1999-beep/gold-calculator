## 一、用户故事

### 一句话概述

个人黄金投资者需要一个便捷的移动端工具来记录交易、实时查看持仓盈亏和金价趋势，以便于高效管理实物黄金投资并辅助交易决策。

### 用户故事三要素

**As a：** 个人黄金投资者（25-50岁，有实物黄金投资习惯）  
**I want to：** 能够随时用手机记录买入/卖出交易，实时查看持仓总览和金价走势图表  
**So that：** 快速掌握投资盈亏情况，基于数据做出买入/卖出决策，避免手工记账的繁琐和错误

---

## 二、背景信息

### 现状（As Is）

1. **记录方式落后**：投资者使用纸质账本、Excel表格或手机记事本记录交易，查询不便且容易丢失
2. **计算效率低下**：手动计算持仓成本、盈亏率费时费力，且容易出错
3. **缺乏趋势参考**：无法直观查看金价历史走势，买卖时机判断靠感觉
4. **数据不安全**：本地文件容易丢失，不支持备份和恢复

### 期望（To Be）

1. **通过移动端H5工具**，投资者可以随时随地记录交易（买入/卖出），系统自动计算持仓成本和盈亏
2. **通过首页持仓总览**，实时展示总克重、均价成本、当前价值、总盈亏、盈亏率，数据一目了然
3. **通过金价趋势图表**，以折线图展示历史金价走势（5个时间维度），辅助交易决策
4. **通过数据导入/导出功能**，保障数据安全，支持JSON格式备份和恢复

## 三、验收标准

### 场景1：正常流程 - 记录买入交易

**Given** 用户持有一定金额的现金，准备买入实物黄金  
**When** 用户在"买入"页面填写交易信息（日期、克重、金价、工费）并点击"保存记录"  
**Then** 系统校验必填项完整且数值合法 → 保存交易记录到LocalStorage → 显示Toast提示"保存成功" → 清空表单等待下次输入

**验收细节：**
- 日期默认今天，可修改
- 克重、金价必填，且必须 > 0
- 工费选填，默认0，必须 ≥ 0
- 备注选填，最长100字符
- 保存成功后，首页持仓总览自动更新

---

### 场景2：正常流程 - 查看持仓盈亏

**Given** 用户已记录至少一笔买入交易  
**When** 用户打开应用，停留在"首页"Tab  
**Then** 系统自动计算并展示持仓总览卡片，包含：
- 总克重（买入总量 - 卖出总量）
- 均价成本（总花费 / 总克重）
- 总花费（买入总支出 - 卖出回款）
- 当前价值（总克重 × 当前卖出价）
- 总盈亏（当前价值 - 总花费）
- 盈亏率（总盈亏 / 总花费 × 100%）

**验收细节：**
- 若未设置金价，当前价值、总盈亏、盈亏率显示"-"
- 盈利时数值显示红色（`#FF4D4F`）
- 亏损时数值显示绿色（`#52C41A`）
- 点击"金价"卡片的"编辑"按钮，可更新买入价和卖出价

---

### 场景3：正常流程 - 查看金价趋势图表

**Given** 用户已多次更新金价（系统自动记录历史金价数据）  
**When** 用户在首页查看"金价趋势"图表，并切换时间维度（实时/近1月/近3月/半年/1年）  
**Then** 系统按选择的时间范围过滤历史金价数据 → 以双折线图展示买入价和卖出价走势 → X轴显示日期，Y轴显示金价（元/克）

**验收细节：**
- 默认显示"实时"维度（今天的数据）
- 折线图使用 Recharts 渲染，支持触摸滑动查看数据点详情
- 买入价折线颜色：金色（`#D4AF37`）
- 卖出价折线颜色：银色（`#C0C0C0`）
- 若无历史数据，显示空状态提示"暂无历史金价数据"

---

### 场景4：正常流程 - 记录卖出交易

**Given** 用户持有可卖出的黄金克重  
**When** 用户在"卖出"页面填写交易信息（日期、克重、金价）并点击"确认卖出"  
**Then** 系统校验卖出克重不超过持仓克重 → 二次确认弹窗 → 确认后保存交易记录 → 显示Toast提示"卖出成功"

**验收细节：**
- 页面顶部显示"可卖克重：XXX克"（买入总量 - 卖出总量）
- 卖出克重必填，且必须满足：> 0 且 ≤ 可卖克重
- 金价必填，且必须 > 0，默认填充当前卖出价
- 系统实时计算"预估收入"（克重 × 金价）
- 若超过可卖克重，提示"卖出克重不能超过持仓克重"

---

### 场景5：异常处理 - 数据存储失败

**Given** 用户尝试保存交易记录  
**When** 浏览器 LocalStorage 存储空间不足或发生异常  
**Then** 系统捕获异常 → 显示错误提示"保存失败，请检查浏览器存储空间" → 不丢失用户已输入的数据（表单保留）

**验收细节：**
- 错误信息用 Toast 或 Modal 弹窗展示
- 表单数据保留，用户可重新尝试保存
- 建议用户及时导出数据备份

---

### 场景6：异常处理 - 金价编辑失败

**Given** 用户在首页点击"编辑"按钮，尝试更新金价  
**When** 系统保存金价到 LocalStorage 时发生错误  
**Then** 系统捕获异常 → 弹窗提示"保存失败，请重试" → 弹窗不关闭，用户可修改后重新保存

**验收细节：**
- 弹窗内显示错误提示
- 用户可修改买入价或卖出价后重新点击"保存"
- 或点击"取消"关闭弹窗

---

### 场景7：边界情况 - 无交易记录时查看持仓

**Given** 用户首次打开应用，无任何交易记录  
**When** 用户停留在首页查看持仓总览  
**Then** 系统显示默认值：总克重 0 克，均价成本 -，总花费 ¥0.00，当前价值 -，总盈亏 -，盈亏率 -

**验收细节：**
- 所有数值字段显示"-"或"0"（根据语义）
- 不报错，界面友好展示空状态
- 引导用户去"买入"页面记录第一笔交易

---

### 场景8：边界情况 - 金价未设置时查看趋势图表

**Given** 用户从未编辑过金价  
**When** 用户停留在首页查看"金价趋势"图表  
**Then** 系统检测到 `priceHistory` 为空 → 显示空状态提示"暂无历史金价数据，记录金价后将显示趋势图"

**验收细节：**
- 空状态包含图标（📊）、标题、说明文字
- 引导用户点击"金价"卡片的"编辑"按钮设置金价
- 设置金价后，自动记录到 `priceHistory`，图表刷新

---

### 场景9：边界情况 - 导入格式错误的JSON文件

**Given** 用户尝试导入数据备份文件  
**When** 用户选择了一个格式不正确的JSON文件（缺少必需字段或JSON语法错误）  
**Then** 系统解析文件失败 → 显示错误提示"导入失败，请检查文件格式是否正确" → 不覆盖本地数据

**验收细节：**
- 校验规则：
  - 文件必须是 valid JSON
  - 必须包含 `records` 数组
  - `records` 中每个对象必须包含 `id`, `type`, `date`, `weight`, `price` 字段
- 若格式错误，提示具体原因（如"缺少records字段"）
- 若格式正确，覆盖本地数据并刷新页面

---

### 场景10：边界情况 - 卖出时持仓为0

**Given** 用户没有任何买入记录，或已全部卖出  
**When** 用户切换到"卖出"页面  
**Then** 系统检测到可卖克重为0 → 显示提示"当前无可卖出的黄金，请先买入" → "确认卖出"按钮置灰不可点击

**验收细节：**
- 可卖克重显示"0克"
- 克重输入框禁用或提示"请先买入黄金"
- 引导用户去"买入"页面记录交易

## 四、功能结构

### 4.1 功能地图

```
黄金管家（黄金持仓盈亏计算器）
├── 首页（Home）
│   ├── 实时金价卡片
│   │   ├── 买入价展示（金色主题）
│   │   ├── 卖出价展示（银色主题）
│   │   └── 编辑按钮（点击弹窗修改）
│   ├── 金价趋势图表
│   │   ├── 时间维度切换（实时/近1月/近3月/半年/1年）
│   │   └── 双折线图展示（买入价/卖出价）
│   └── 持仓总览卡片
│       ├── 总克重
│       ├── 均价成本
│       ├── 总花费
│       ├── 当前价值
│       ├── 总盈亏（金额）
│       └── 盈亏率（百分比）
│
├── 买入（Buy）
│   ├── 日期选择器（默认今天）
│   ├── 克重输入（必填，> 0）
│   ├── 金价输入（必填，> 0，默认上次金价）
│   ├── 工费输入（选填，≥ 0，默认0）
│   ├── 备注输入（选填，最长100字符）
│   └── 保存记录按钮
│
├── 卖出（Sell）
│   ├── 可卖克重提示
│   ├── 日期选择器（默认今天）
│   ├── 克重输入（必填，> 0 且 ≤ 可卖克重）
│   ├── 金价输入（必填，> 0，默认当前卖出价）
│   ├── 预估收入展示（实时计算）
│   └── 确认卖出按钮
│
├── 记录（Records）
│   ├── 交易记录列表（按日期倒序）
│   │   ├── 买入记录（绿色标签 + 左侧绿色竖线）
│   │   └── 卖出记录（红色标签 + 左侧红色竖线）
│   └── 删除记录（左滑或长按 → 二次确认）
│
└── 设置（Settings）
    ├── 数据导出（生成JSON文件并下载）
    ├── 数据导入（选择JSON文件并解析）
    └── 清空数据（二次确认，输入"确认清空"）
```

## 五、非功能需求

### 5.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 首屏加载时间 | < 1秒（4G网络） | 从打开页面到首屏渲染完成 |
| 页面切换响应 | < 200ms | 点击底部Tab到页面渲染完成 |
| 数据保存响应 | < 100ms | 点击"保存"到LocalStorage写入完成 |
| 图表渲染时间 | < 500ms | 切换时间维度到图表渲染完成 |

### 5.2 兼容性要求

| 平台 | 版本要求 | 说明 |
|------|----------|------|
| iOS Safari | iOS 12+ | 支持iPhone 6及以上 |
| Android Chrome | Android 8+ | 支持大部分安卓机型 |
| 微信内置浏览器 | 支持 | 微信生态内正常使用 |
| 支付宝内置浏览器 | 支持 | 支付宝生态内正常使用 |

### 5.3 数据安全性

- 数据存储在用户本地（LocalStorage），不上传云端
- 不支持多设备同步（V3版本考虑云端同步）
- 用户可自行导出JSON备份，保障数据安全
- 导入数据时覆盖本地数据，需二次确认

### 5.4 离线使用

- ✅ 完全支持离线使用，所有功能无需联网
- ✅ 数据存储在LocalStorage，离线可读写
- ❌ 不支持金价自动获取（V3版本考虑接入API）

## 七、附录

### 7.1 名词解释

| 名词 | 解释 | 计算公式（如适用） |
|------|------|-------------------|
| 克重 | 黄金的重量，单位：克（g） | - |
| 金价 | 每克黄金的价格，单位：元/克 | - |
| 工费 | 购买黄金首饰时的工艺费，单位：元 | - |
| 总花费 | 买入黄金时的总支出 | 克重 × 金价 + 工费 |
| 均价成本 | 平均每克黄金的成本 | 总花费 / 总克重 |
| 当前价值 | 按当前卖出价估算的持仓价值 | 总克重 × 当前卖出价 |
| 总盈亏（金额） | 盈利或亏损的金额 | 当前价值 - 总花费 |
| 盈亏率（百分比） | 盈亏占总花费的比例 | (总盈亏 / 总花费) × 100% |
