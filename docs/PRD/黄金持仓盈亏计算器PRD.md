# 黄金持仓盈亏计算器 - 产品需求文档(PRD)

**文档版本：** V2.0
**撰写日期：** 2026-05-31
**撰写人：** 产1品经理
**产品名称：** 黄金持仓盈亏计算器（黄金管家）

---

## 一、背景

### 1.1 行业背景
随着国际金价持续波动，越来越多的个人投资者选择通过实物黄金（金条、金饰等）进行资产配置。然而，当前市场上缺乏简单易用的个人黄金持仓管理工具，投资者难以实时掌握自己的持仓成本、盈亏情况。

### 1.2 用户痛点
1. **记录分散**：多数用户使用纸质账本、Excel或手机记事本记录交易，查询不便
2. **计算困难**：手动计算持仓成本、盈亏率容易出错
3. **缺乏趋势**：无法直观看到金价历史走势，影响交易决策
4. **数据丢失**：本地文件容易丢失，不支持云端同步

### 1.3 产品定位
面向个人黄金投资者的移动端H5工具，提供**简单、直观、实用**的持仓管理和盈亏计算功能。

## 二、用户故事

### 2.1 目标用户画像
- **年龄：** 25-50岁
- **职业：** 白领、中小企业主、自由职业者
- **投资习惯：** 实物黄金（金条、金饰）、分散投资
- **技术水平：** 基础互联网使用能力
- **核心需求：** 快速记录交易、实时查看盈亏

### 2.2 典型使用场景

#### 场景1：买入黄金后记录
> **作为** 黄金投资者  
> **我想要** 快速记录买入的克重、金价、工费  
> **以便于** 后续计算持仓成本和盈亏

**使用流程：**
1. 用户在金店买完黄金
2. 打开应用，点击"买入"Tab
3. 输入日期、克重、金价、工费
4. 点击保存，记录成功

#### 场景2：查看当前持仓盈亏
> **作为** 黄金投资者  
> **我想要** 在首页看到当前持仓的总克重、成本、盈亏  
> **以便于** 快速了解投资收益情况

**使用流程：**
1. 打开应用，自动显示首页
2. 查看持仓总览卡片（总克重、均价成本、总盈亏）
3. 若金价变动，点击"金价"卡片编辑最新金价
4. 持仓盈亏自动更新

#### 场景3：卖出黄金前预估收入
> **作为** 黄金投资者  
> **我想要** 在卖出前预估能收回多少资金  
> **以便于** 决策是否卖出

**使用流程：**
1. 点击"卖出"Tab
2. 输入卖出克重、当前金价
3. 系统自动显示预估收入和盈利
4. 确认后保存卖出记录

#### 场景4：查看金价趋势
> **作为** 黄金投资者  
> **我想要** 查看金价的历史走势  
> **以便于** 判断买入/卖出时机

**使用流程：**
1. 在首页查看"金价趋势"图表
2. 切换时间维度（实时/近1月/近3月/半年/1年）
3. 观察买入价和卖出价的双线走势
4. 辅助交易决策

## 三、产品目标

### 3.1 核心目标
1. **降低记录门槛**：3步完成交易记录，30秒内完成操作
2. **提升决策效率**：实时查看持仓盈亏，辅助交易决策
3. **数据安全保障**：本地存储 + 导入导出，防止数据丢失

### 3.2 量化指标（KPI）
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 记录完成时间 | < 30秒 | 从打开到保存完成 |
| 数据准确率 | 100% | 盈亏计算准确性 |
| 页面加载速度 | < 1秒 | 首屏渲染时间 |
| 用户留存率 | > 60% | 次月留存 |

### 3.3 非目标
- ❌ 不支持黄金期货、纸黄金等金融产品
- ❌ 不提供投资理财建议
- ❌ 不接入交易功能（仅记录工具）
- ❌ 不支持多用户协作

## 五、功能详述

### 5.1 首页（Home）

#### 5.1.1 实时金价卡片

**功能描述：**
展示当前买入价和卖出价，支持手动编辑更新。

**功能详情：**
| 字段 | 说明 | 规则 |
|------|------|------|
| 买入价 | 当前黄金买入参考价 | 单位：元/克；未设置时显示"未设置" |
| 卖出价 | 当前黄金卖出参考价 | 单位：元/克；未设置时显示"未设置" |
| 编辑按钮 | 点击弹出编辑弹窗 | 同时更新买入价和卖出价 |

**交互逻辑：**
1. 点击"编辑"按钮 → 弹出 Modal 弹窗
2. 弹窗内输入买入价、卖出价
3. 点击"保存" → 更新金价 + 自动记录到历史
4. 点击"取消" → 关闭弹窗，不保存

**视觉设计：**
- 买入价：金色主题（`#D4AF37`）
- 卖出价：银色主题（`#C0C0C0`）
- 卡片式设计，左右并排展示

---

#### 5.1.2 金价趋势图表

**功能描述：**
以折线图形式展示历史金价走势，支持5个时间维度切换。

**功能详情：**
| 维度 | 时间范围 | 数据说明 |
|------|----------|----------|
| 实时 | 今天 | 当天记录的所有金价 |
| 近1月 | 最近30天 | 每日最后一条记录 |
| 近3月 | 最近90天 | 每日最后一条记录 |
| 半年 | 最近180天 | 每周最后一条记录 |
| 1年 | 最近365天 | 每月最后一条记录 |

**交互逻辑：**
1. 默认显示"实时"维度
2. 点击时间维度标签 → 切换数据范围
3. 图表支持触摸滑动查看详情
4. 点击数据点 → 显示具体日期和价格

**视觉设计：**
- 买入价折线：金色（`#D4AF37`）
- 卖出价折线：银色（`#C0C0C0`）
- X轴：日期；Y轴：金价（元/克）
- 图表高度：200px

**数据来源：**
- 用户每次编辑金价时，自动记录到 `priceHistory`
- 用户可手动添加历史金价（V3版本功能）

---

#### 5.1.3 持仓总览卡片

**功能描述：**
展示当前持仓的核心数据，基于实时卖出价计算盈亏。

**功能详情：**
| 字段 | 说明 | 计算公式 |
|------|------|----------|
| 总克重 | 当前持有的黄金克重 | 买入总量 - 卖出总量 |
| 均价成本 | 平均每克黄金的成本 | 总花费 / 总克重 |
| 总花费 | 净投入金额 | 买入总花费 - 卖出回款 |
| 当前价值 | 按当前卖出价估算的价值 | 总克重 × 当前卖出价 |
| 总盈亏（金额） | 盈利或亏损金额 | 当前价值 - 总花费 |
| 盈亏率（百分比） | 盈亏占总花费的比例 | (总盈亏 / 总花费) × 100% |

**交互逻辑：**
1. 每次打开首页自动计算
2. 金价更新后自动刷新
3. 盈亏金额和盈亏率用颜色区分：
   - 盈利：红色（`#FF4D4F`）
   - 亏损：绿色（`#52C41A`）

**视觉设计：**
- 2×3 网格布局
- 数值加粗显示
- 盈亏用大号字体突出

---

### 5.2 买入（Buy）

#### 5.2.1 记录买入表单

**功能描述：**
录入买入黄金的交易记录。

**功能详情：**
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 日期 | 日期选择器 | 是 | 默认今天；可修改 |
| 克重 | 数字输入 | 是 | 单位：克；支持小数 |
| 金价 | 数字输入 | 是 | 单位：元/克；默认上次金价 |
| 工费 | 数字输入 | 否 | 单位：元；默认0 |
| 备注 | 文本输入 | 否 | 最长100字符 |

**交互逻辑：**
1. 填写表单 → 实时计算总花费（克重 × 金价 + 工费）
2. 点击"保存记录" → 校验必填项 → 保存到 LocalStorage
3. 保存成功 → 清空表单，显示 Toast 提示
4. 保存失败 → 显示错误提示（如存储空间不足）

**校验规则：**
- 克重：> 0
- 金价：> 0
- 工费：≥ 0

**视觉设计：**
- 表单式布局
- 输入框带单位提示（克/元/元/克）
- 保存按钮固定底部

---

### 5.3 卖出（Sell）

#### 5.3.1 记录卖出表单

**功能描述：**
录入卖出黄金的交易记录，自动计算预估收入。

**功能详情：**
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 可卖克重提示 | 文本 | - | 显示当前可卖出的总克重 |
| 日期 | 日期选择器 | 是 | 默认今天；可修改 |
| 克重 | 数字输入 | 是 | 单位：克；最大不超过可卖克重 |
| 金价 | 数字输入 | 是 | 单位：元/克；默认当前卖出价 |
| 预估收入 | 文本 | - | 自动计算：克重 × 金价 |

**交互逻辑：**
1. 进入页面 → 自动计算可卖克重并显示
2. 输入克重 → 实时校验是否超过可卖克重
3. 输入金价 → 实时计算预估收入
4. 点击"确认卖出" → 二次确认弹窗 → 保存记录
5. 保存成功 → 清空表单，显示 Toast 提示

**校验规则：**
- 克重：> 0 且 ≤ 可卖克重
- 金价：> 0
- 若超过可卖克重 → 提示"卖出克重不能超过持仓克重"

**视觉设计：**
- 可卖克重用醒目颜色提示（金色）
- 预估收入用大号字体显示
- 确认按钮用红色强调

---

### 5.4 记录（Records）

#### 5.4.1 交易记录列表

**功能描述：**
展示所有买入和卖出记录，支持删除操作。

**功能详情：**
| 字段 | 说明 |
|------|------|
| 交易类型 | 买入（绿色标签）/ 卖出（红色标签） |
| 日期 | 交易日期 |
| 克重 | 交易克重（带单位"克"） |
| 金价 | 交易金价（带单位"元/克"） |
| 总花费/收入 | 买入显示总花费；卖出显示预估收入 |
| 备注 | 若有备注则显示 |

**交互逻辑：**
1. 列表按日期倒序排列（最新在前）
2. 左滑或长按记录 → 显示"删除"按钮
3. 点击"删除" → 二次确认弹窗 → 删除记录
4. 删除成功 → 列表刷新，显示 Toast 提示

**视觉设计：**
- 买入记录：左侧绿色竖线标识
- 卖出记录：左侧红色竖线标识
- 列表卡片式设计
- 删除按钮用红色

---

### 5.5 设置（Settings）

#### 5.5.1 数据导出

**功能描述：**
将所有交易记录和设置导出为JSON文件。

**交互逻辑：**
1. 点击"导出数据" → 生成JSON文件
2. 自动下载到本地（文件名：`gold_records_YYYYMMDD_HHMMSS.json`）
3. 导出成功 → 显示 Toast 提示"导出成功"

**导出数据格式：**
```json
{
  "records": [...],
  "settings": {...},
  "exportTime": "2026-05-31T12:00:00.000Z"
}
```

---

#### 5.5.2 数据导入

**功能描述：**
从JSON文件导入交易记录和设置。

**交互逻辑：**
1. 点击"导入数据" → 选择JSON文件
2. 解析文件 → 校验格式是否正确
3. 格式正确 → 覆盖本地数据 → 显示 Toast 提示"导入成功"
4. 格式错误 → 显示错误提示"文件格式不正确"

**校验规则：**
- 文件必须是 valid JSON
- 必须包含 `records` 数组
- `records` 中每个对象必须包含 `id`, `type`, `date`, `weight`, `price` 字段

---

#### 5.5.3 清空数据

**功能描述：**
清空所有本地数据（谨慎操作）。

**交互逻辑：**
1. 点击"清空数据" → 二次确认弹窗（输入"确认清空"）
2. 确认 → 清空 LocalStorage → 显示 Toast 提示"已清空"
3. 取消 → 关闭弹窗，不操作

## 八、附录

### 8.1 名词解释
| 名词 | 解释 |
|------|------|
| 克重 | 黄金的重量，单位：克（g） |
| 金价 | 每克黄金的价格，单位：元/克 |
| 工费 | 购买黄金首饰时的工艺费，单位：元 |
| 总花费 | 买入黄金时的总支出（克重 × 金价 + 工费） |
| 均价成本 | 平均每克黄金的成本（总花费 / 总克重） |
| 当前价值 | 按当前卖出价估算的持仓价值（总克重 × 当前卖出价） |
| 盈亏 | 当前价值 - 总花费 |
| 盈亏率 | 盈亏 / 总花费 × 100% |
